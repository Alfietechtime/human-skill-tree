## 篇1：项目诞生篇

**标题：** 一个人做了个AI教育产品，从0到上线的全过程

**封面文案建议：** 代码编辑器截图 + "独立开发者手记" 大字，暗色调

---

花了两周，一个人从零做了一个 AI 学习平台。

起因很简单：2025 年 AI 有了 Skill、MCP、Tool Use，等于有了自己的技能树。但人类学东西的方式还是老一套——看视频、划重点、考前突击。

PNAS 顶刊 2025 年发了一篇：用 GPT-4 辅导高中生，数学成绩涨了 48%-127%。但有个前提——必须加教学设计，否则学生直接抄 AI 答案，反而变差。

所以核心问题不是"AI 能不能教"，而是"怎么让 AI 正确地教"。

我的方案：写了 33 个 AI Agent Skill，覆盖 K-12 到职业发展到人情世故，每个技能内置认知科学方法论（间隔重复、主动回忆、苏格拉底提问）。然后套了一层 Next.js 16 的 Web 壳，做成交互式平台。

技术栈：
- Next.js 16 + TypeScript + Tailwind CSS v4
- Vercel AI SDK v6 + OpenRouter（18个模型）
- Supabase Auth + PostgreSQL
- React Flow 技能树可视化
- Cloudflare CDN（国内免翻可访问）

最有意思的部分是"苏格拉底导师系统"——6 个 AI 角色，各有性格和教学风格。导师之间共享记忆，你从 Aria 切到 Marcus，Marcus 知道你之前学了什么、在哪卡壳。课后导师们还会在群聊里讨论你的表现。

完全开源，AGPL-3.0。

在线体验：humanskilltree.yechengzhang.com
GitHub：github.com/24kchengYe/human-skill-tree-app

## 篇2：技术复盘篇

**标题：** 独立开发一个AI学习平台的技术决策复盘

**封面文案建议：** 架构图或终端截图，"Tech Stack & Architecture" 标题

---

复盘一下这个项目里的关键技术决策。

**1. 7 层 System Prompt 架构**

每次 AI 对话构建 7 层提示词：导师人格 → 故事背景 → 苏格拉底约束 → 跨导师记忆 → 学习者角色 → 技能方法论 → KP 追踪。每层可独立替换和测试。

**2. AI 隐藏标签协议**

AI 在回复末尾嵌入 HTML 注释标签：`<!--KP: 概念1 | 概念2-->`。前端自动解析，更新知识点和掌握度。用户完全看不到。比 JSON mode 轻量，兼容所有模型。

**3. 跨导师记忆**

每个导师回复带 `<!--TAUGHT:-->` `<!--STUCK:-->` `<!--ATTITUDE:-->` 标签。按 `导师×技能` 维度存储。切换导师时注入上下文（限 300 tokens），新导师自然接上话。

**4. 付费模型分级**

18 个模型分 4 层（Free/Basic/Pro）。Middleware 检测计划等级，API 层二次校验。付费走 LemonSqueezy Webhook 自动升级，前端 60s 轮询 + tab 可见时刷新。

**5. 国内可访问**

Cloudflare CNAME 代理 + Vercel，SSL Full (strict)。Middleware 检测 `cf-ipcountry` header，中国 IP 自动跳转中文界面。

**6. 双仓库策略**

公开仓库放阉割版（不含付费代码），私有仓库放完整版。Vercel CLI 部署，不走 Git webhook。

踩的坑：
- OpenRouter 不完全支持 Responses API，必须强制 Chat Completions
- Turbopack 把中文引号 `"..."` 当 JS 字符串解析，改用 `「...」`
- Cloudflare 代理 + Vercel 如果 SSL 设置不是 Full (strict) 会无限重定向

## 篇3：认知科学篇

**标题：** 为什么你用AI学习没效果？从认知科学角度拆解

**封面文案建议：** 简洁的对比图——"直接问AI" vs "结构化AI学习"，暗色调

---

大多数人用 AI 学习的方式是：打开 ChatGPT，问一个问题，看答案，觉得自己懂了。

这就是认知科学里说的 fluency illusion（流畅性错觉）——你看懂了，但什么都没记住。

几个关键研究结论：

**1. 最常用的学习方法，效果最差**

Dunlosky 的元分析（242 项研究，16.9 万人）：反复阅读、划重点、做摘要，效果排最后。真正有效的是检索练习（合上书默写）和间隔重复。

**2. AI 辅导需要教学设计才有效**

PNAS 2025：GPT-4 辅导让成绩涨 48%-127%，但前提是加了"给提示不给答案"的教学约束。不加约束，学生直接抄答案，成绩反而下降。

**3. 算法优化的复习间隔 > 人工设定**

PNAS 2019，1200 万 Duolingo 用户数据：算法优化的间隔重复显著优于手动设定的复习计划。

基于这些研究，我在 Human Skill Tree 里做了几件事：

- **苏格拉底式提问**：AI 不直接给答案，先用问题引导你思考
- **间隔重复**：Leitner 箱式系统自动安排复习（1d→3d→7d→14d）
- **知识点自动提取**：AI 从对话中提取原子概念，追踪掌握度
- **多导师切换**：不同教学风格匹配不同学习者偏好

不是"让 AI 更聪明"，而是"让 AI 正确地教"。

## 篇4：人情世故篇（破圈）

**标题：** 做了个AI教人情世故的功能，意外成了最受欢迎的技能

**封面文案建议：** 终端/对话截图 + "AI × 中国社交" 标题

---

做 Human Skill Tree 这个项目时，本来重心在学科教育——数学、编程、科研方法论。

结果上线后，用得最多的技能是"人情世故"。

这个技能覆盖的场景：

**饭局**：座位怎么排（面门为尊），敬酒顺序（等领导先敬完再逐一敬），杯子高度（低于对方），什么时候该抢买单。

**送礼**：禁忌速查——钟（终）、梨（离）、伞（散）、鞋（邪）。红包带 8 避 4，金额看关系远近。

**职场**：多请示多汇报，不越级汇报，"考虑考虑" = "不行"，"有空聊聊" = "要谈正事了"。

关键能力：AI 可以模拟场景。你告诉它"我是部门最年轻的，要和客户吃饭"，它就扮演客户/领导，让你练怎么接话、怎么应对。

这些东西没有任何教材教过，但每个人都在真实场景里吃过亏。特别是从小城市到大城市、从学校到职场的过渡期。

技术上是同一套苏格拉底导师系统，但这个技能的 SKILL.md 里写了大量中国特定的社交规则和场景模板。
